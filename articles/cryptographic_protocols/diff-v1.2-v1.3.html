Differences between TLSv1.2 and TLSv1.3



Here are the highlights of the differences:

Efficiency:
Handshake RTTs (Round Trip Time) have been reduced from 2 to 1:
Faster RTTs result in faster connections between the client and the server. This in turn results in faster load times for content (for example, faster load time for webpages due to HTTPS). The figure below helps in clarifying the idea:


Introduction of 0-RTT Session Resumption
Servers that negotiate a TLSv1.3 connection will remember the sessions they have made. With reduced load times, session resumptions are possible. Security:
A large part of security for TLSv1.3 comes from removing support for old and weak (provably weak) security techniques. These are:

SHA-1 - Proven to be non-collison resistant. This weakness allows attackers to create multiple, differing input sources that, when the MD5 algorithm is used, result in the same output fingerprint.
RC4 - Not secure for use in HTTPS
DES -
3DES - Removed by NIST in IPsec and SSL. Provable weakness due to its size.
AES-CBC - Responsible for attacks such as BEAST
MD5 - Proved to be non secure because of being non-collison resistant. This weakness allows attackers to create multiple, differing input sources that, when the MD5 algorithm is used, result in the same output fingerprint.
Arbitrary Diffie-Hellman groups — CVE-2016-0701
EXPORT-strength ciphers – Responsible for FREAK and LogJam
