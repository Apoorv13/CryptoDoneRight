TLS (Transport Layer Security) Protocol

Last Updated Thu, 21 Aug 2017 12:00:01 -0400

Obsoletes SSL

TLS (Transport Layer Security) is a cryptographic communications protocol that can be used to provide privacy, message integrity, and authentication of the communicating parties in network communications. TLS is unfortunately sometimes referred to as "SSL," the obsoleted, and vulnerable algorithm it replaced. The current version of TLS is blah and is specified in RFC-.

TLS is a critical component, if not the critical component, of contemporary web security. Every "secure" web session (i.e., "https://") relies on either TLS or (if poorly configured) SSL. Therefore, it is essential that administrators, and other relevant IT professionals, correctly configure their TLS installations. Although TLS is its own protocol, it relies on a number of component protocols for bulk encryption, authentication, and so forth. Selecting the appropriate algorithms and algorithm parameters is an important part of configuration.

Best Practices

Do:

  • Use TLS 1.2 and prepare to move to TLS 1.3
  • Carefully choose certificate generation including Certificate Authority choice
  • Secure private keys
  • Choose algorithms appropriate for the context and the TLS version
  • Secure random number generation
  • Test TLS to ensure it is working as expected

Don't:

  • Use SSL under any circumstances
  • Use TLS 1.0 unless absolutely necessary
  • Use SHA-1 with your Digital Certificates
  • Assume that just "using" TLS makes your communications secure
