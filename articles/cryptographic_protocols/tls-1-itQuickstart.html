Upgrade/Patch Management

Concept: It is important to understand that the fixes for both the vulnerabilities is to ensure that any services or devices that you own/control and accepts TLS connections should not do so on TLSv1.0. Please bear this in mind for whatever fix maybe applicable. Click here for basic understanding of the TLS packet flow.

Web Servers:
Completely disable TLS1.0 on the server. The recommendation is to move to TLSv1.2.
  • Nginx Web Server

    To disable TLSv1.0 in the Nginx web server, you can use the SSL_protocols directive. This will be located in the server or http blocks in your configuration. For instance, on Ubuntu, you can either add this globally to /etc/nginx/nginx.conf inside of the httpblock, or to each server block in the /etc/nginx/sites-enabled directory. 
    $sudo nano /etc/nginx/nginx.conf
    To disable TLSv1.0, your TLS_protocols directive should be set like this: 
    
...
TLS_protocols TLSv1.1 TLSv1.2;
...
    You should restart the server after you have made the above modification: 
    sudo service nginx restart

  • Apache Web Server

    To disable TLSv1.0 on the Apache web server, you will have to adjust the TLSProtocol directive provided by the mod_TLS module. This directive can be set either at the server level or in a virtual host configuration. Depending on your distribution's Apache configuration, the TLS configuration may be located in a separate file that is sourced.

    On Ubuntu, the server-wide specification for servers can be adjusted by editing the /etc/apache2/mods-available/TLS.conf file. If mod_TLS is enabled, a symbolic link will connect this file to the mods-enabledsubdirectory: 
    $sudo nano /etc/apache2/mods-available/ssl.conf

    On CentOS, you can can adjust this in the TLS configuration file located here (if TLS is enabled): 
    $sudo nano /etc/httpd/conf.d/ssl.conf
    Inside you can find the TLSProtocol directive. If this is not available, create it. Modify this to explicitly remove support for TLSv3: 
    
...
SSLProtocol all -SSLv3 -SSLv2
...

    Save and close the file. Restart the service to enable your changes.
    On Ubuntu, you can type: 
    
$sudo service apache2 restart
    On CentOS, this would be: 
    $sudo service httpd restart


Browsers
Upgrade browser (client) to the latest version or if for whatever reason you need to use an older version be sure to disable TLS 2.0 and TLS 3.0 Protocols. Most up to date browsers/servers use TLS_FALLBACK_SCSV. If a client requests a TLS protocol version which is lower than the highest supported by the server (and client), the server will identify intentional downgrade fallback and will drop the connection.

Here is a list of some common browsers and if whether the mitigations (no support for TLSv3.0) are completed.

  • Google Chrome: On the browser address bar, visit chrome://flags and under "Minimum SSL/TLS version supported.", change from "Default" to "TLS 1.1".)
  • Mozilla Firefox: In the addresa bard, type "about:config" and hit enter. Post this, in the "Search" field, enter "tls". Find and double-click the entry for "security.tls.version.min" and set the interger value to "2" to force a minimum protocol of TLSv1.1 (entering "3" would force TLSv1.2). Do not forget to clear your cache post this change.
  • Internet Explorer: Left click the gear icon on the top-left of the browser window. Select "Internet Options" from the dropdown menu. Next, click the “Advanced” tab, scroll down and deselect “SSL 3.0” and “TLS 1.0”. Finally, click OK (you may need to refresh your browser post these changes).
  • Opera: Start by clicking on Ctrl + F12. Scroll down to the Network section and click on Change proxy settings.... Post this, select the "Advanced" tab and scroll down to Security category and manually check the option box for Use TLS 1.1 and Use TLS 1.2.
  • Safari: There are no options for enabling SSL protocols. If you are using Safari version 7 or greater, TLS 1.1 and TLS 1.2 are automatically enabled.


VPN:
VPN connections can be made using TLS. Care must be taken that any VPN gateways (that accept the connection from VPN clients) do not have TLSv3.0 configured. There cannot be any support for legacy protocols and all VPN connections should be based on TLSv1.2.
